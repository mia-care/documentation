Version 2.4 Release Notes

February 10th, 2026

The minor release v2.4 brings improvements of the UX/UI, AI evaluation and document management.

AI & Quality Assurance​

  • AI Quality Scoring: Improve accuracy of AI evaluation for requirements and tests to ensure more realistic "Clarity" and "Coherence" metrics.
  • Test Quality Insights: Integrate specific suggestions to warn users when test execution modes are missing or when tests need manual execution.
  • Requirement Quality Table: Users can now sort requirement tables based on AI-generated quality score.

Software Item (SWI) Management​

  • Non-Semantic Versioning: SWI versions now support non-semantic formats. The system provides an insight if a non-semantic version is used to ensure it isn't a design error.
  • SWI Design Enforcement: Added "Error-level" insights that trigger when mandatory architecture designs are missing for SWIs with Safety Class B or C.
  • SBOM Enhancements: Licenses are now detected and included in the Software Bill of Materials (SBOM) file during dependency detection.

Documentation & Traceability​

  • Docx Compatibility: Documentation templates now support interpolated data export to Word (.docx), ensuring column sizes and headers are correctly rendered for professional readers.
  • Traceability Matrix: Links between test entities and specific code implementations (e.g. Git URLs) are now more visible for auditing and coverage analysis.
  • Read-Only History: Previous versions of documentation templates are now presented in a dedicated history view in READ-ONLY mode to prevent accidental modifications to legacy data.

Visualization & Usability​

  • Frozen Released Versions: To maintain data integrity, the system now warns users and prevents any active modifications once a version is marked as "Released".
  • Test Dashboards: New views to distinguish between "Current Version Tests" and "Total Valid Tests" (including regression tests) to provide a better snapshot of the testing scope.
  • Risk Table Refactoring: Risks now include linked vulnerabilities, SWI, and Changes in a single consolidated table view.

System Information​

  • ALM Accessibility: A new tab shortcut is available for users to reach external ALM tools directly from the P4SaMD interface.
